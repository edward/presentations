Download the Data

  1. Download the file here: Singapore_population_data 2011_by_Zone.csv.

Upload Data to Fusion Tables

  1. Go to Google Drive
  2. If you have not already created a Google account, create a Google account and then sign in. Do NOT use your hyper island account or a business account. Fusion Tables is only available from personal gmail accounts.
  3. Select Create -> Fusion Table (If it's not there, you have to press "Connect more apps" and search for fusion tables)

  1. Click Choose File.
  1. Browse to find the XML file that you downloaded.
  2. Click Next >>.
  3. Click Next >> again.
  4. Update the Table name to Singapore Population Data by Zone 2011. Add any attribution data and/or change the description.


  5. Click Finish.

Merge tables

Now that you have the population data in Fusion Tables, we're going to want to visualize it. But the data we have doesn't actual have any actual addresses....All it has is the name of areas. What we can do is find another fusion table that contains the shapes of the areas, and then merge both of them together.

Check out this table here.

It contains the shapes of all planning areas in Singapore. Fusion Tables has special handling for KML (Keyhole Markup Language) that is used to represent geographical features such as points, lines and polygons / boundaries. The table containing areas has two columns we are interested in: one for the name of the area labeled "name", and one labeled "geometry" where each cell has KML for the corresponding boundary.

  1. Make sure you have the Population table you created open in your browser.
  2. In the Population table, select File -> Merge.
  3. paste the following URL in the Text box under Or paste a web address here:: https://www.google.com/fusiontables/DataSource?docid=1eQvVXJSwwJTISJ4C2FDz99w3ArYUjwXHn97-6nI
  4. Click the Next button.
  5. Select the columns you want to merge on. Make sure the dropdown on the left is set to  DGP Zone/Subzone , and the dropdown on the right is set to  name . These columns both have names of areas in the same format. Merge works by looking for exact matches in the selected columns and then linking matching rows together.
  6. Press Next

  7. (Here is where you can optionally discard columns. We don't care about that right now). Click Merge
  8. Select View table. You are now brought to a new table that has all the data merged in. Hooray!
  9. At the top of the page there is a little red + icon. Click it, and press Add map
  10. You now see a sweet map with the boundaries of Singapore highlighted. Feel free to zoom and move around. Click on one of the polygons on the map to see what is displayed in the InfoWindow.

Configure the InfoWindow

Now let's customize the content in the InfoWindow.

  1. On the left hand column, click on Change info window layout....
  2. Select the Custom tab.


  3. This lets you update what is shown when you click on an area. This is pure HTML. You can play around and format the infowindow as you please.

Styling

Now add some styling to the map to create an interesting visualization of the data.

  1. Click On the left sidebar, select Tools -> Change Feature Styles.
  2. Select Fill color under Polygons in the left column.
  3. Select the Gradient tab.
  4. Select the radio button next to Show a gradient.
  5. In the drop-down menu next to Column, select Total Population.
  6. Click on the 'use this range' link



  7. Select any colors you want in the drop-down menus or keep the default.
  8. Click Save when done.
  9. The map will be updated with the new style.

Filtering

Let's say we only want to display areas with more than 100 000 people.

  1. Click the Filter button at the top left of the map
  2. Select "total population" as the property you are filtering on.
  3. Play around with the range and see how it affects the map.

Sharing and Permissions

New tables are automatically set to Private. The visibility needs to be changed to Unlisted or Public to embed in a web page.
  1. Click  the Share button in the top right corner.
  2. Selec t the link that says Change... under the "Who has access" section. Select  Anyone with the link or Public, and then press Save. The visibility will be updated immediately on selection.

  3. Close the window by pressing Done.

Embedding a Fusion Tables Map Visualization in an HTML page

As an alternative to viewing a map in the Fusion Tables application, it is possible to embed a view of a Fusion Table based map visualization in an HTML page.
  1. Create a new html file on your computer and then inside FusionTables, go to Tools-> Publish, and copy the line under "Paste HTML to embed in a website". Paste that in your html file, save, and then open the html file in your browser. Bravo!

Advanced: Using the Google Maps API FusionTablesLayer to Display the map

A more advanced way of embedding a Fusion Tables map into a web page is to use the Fusion Table layer in Google MAPs API. This requires some experience with javascript but has the advantage of allowing you to take advantage of all the features of Google MAPs API. 
  1. Go back to your Fusion Table, and go to the "ROWS" view (not the map one). Hover over the "Total Population" column name, and a little arrow should pop up. Click on it, and go "Change". Rename the table to 'population'.
  2. Go back to the map view you made, and press Tools -> Publish... again.
  3. Instead of selecting the embedding stuff, click on the Get HTML and Javascript part. Select all the code in there.
  4. Inside your code editor, create a new file (File -> New File). Paste the code that you copied from Fusion tables in there.
  5. Save the file. Call it map.html
  6. Open the file in your browser. You should see the map
  7. Now, go back to your code. Update the query sent to the Fusion Table Layer to include a filter. To do so modify "where" clause below, at line 43 as follows:
    8. query: {
      select: 'geometry',
      from: tableid,
      where: 'population > 100 000'
    },

    (Don't worry if the "from" and "select" lines look different from the ones above. You just care about the line that has "where" on it)
  8. Refresh the page in the browser, and you should now only see areas that have population counts over 100 000. Feel free to explore!
